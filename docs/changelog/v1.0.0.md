## 🔐 Security

The system is designed with a **security-by-design** approach, applying multiple layers of protection across authentication, request handling, data storage, and infrastructure.

### 1. Authentication & Authorization

- **JWT-based authentication** using:
  - Access Token (120 seconds)
  - Refresh Token (30 days)
- Passwords are **hashed with salt using bcrypt** before storage.
- **Email verification** is required before account activation.
- Authorization middleware:
  - `RequireAuth` to protect authenticated endpoints.
  - `RequireAdmin` to enforce admin-level access control.

---

### 2. CSRF Protection

- CSRF tokens are **generated and validated using constant-time comparison**.
- Tokens are stored in **HttpOnly + Secure cookies**.
- CSRF validation is enforced on all:
  - `POST`
  - `PUT`
  - `DELETE`
  - `PATCH` requests.

---

### 3. Security Headers

- Standard HTTP security headers are enabled:
  - `X-XSS-Protection`
  - `X-Frame-Options`
  - `X-Content-Type-Options`
- **Strict-Transport-Security (HSTS)** is enforced in production.
- **Content-Security-Policy (CSP)** is explicitly defined to restrict resource sources.
- `Referrer-Policy` is configured to prevent sensitive data leakage.

---

### 4. CAPTCHA Protection

- **Cloudflare Turnstile** is integrated for sensitive actions:
  - User registration
  - User login
- CAPTCHA tokens are **verified server-side**.

---

### 5. Rate Limiting

- **IP-based rate limiting** is implemented.
- In-memory limiter includes a **periodic cleanup mechanism** (every 30 seconds).
- Maximum **50,000 entries** to prevent memory exhaustion.
- Different limits are applied:
  - User routes: `5 requests/second`
  - Admin routes: `50 requests/second`

---

### 6. CORS

- Only **whitelisted origins** defined via environment variables are allowed.
- **Credential sharing** (cookies, authorization headers) is strictly controlled.

---

### 7. Database Security

- Uses **ORM (GORM)** to reduce the risk of SQL injection.
- **Prepared statements** are handled automatically during query execution.

---

### 8. File Upload Security

- File upload validation includes:
  - Maximum file size: **10MB**
  - Allowed file types: `.jpg`, `.jpeg`, `.png`, `.webp`, `.mp4`, `.mov`, `.avi`, `.webm`
- Uses **Presigned URL** strategy for S3-compatible uploads to:
  - Avoid exposing direct upload endpoints
  - Reduce server attack surface
- Filenames are **sanitized** and stored as `UUID + extension`.

---

### 9. Session Management

- Session idle timeout detection (60 minutes).
- Automatic logout on session expiration.
- Secure **token refresh mechanism**.

---

### 10. Frontend Security

- **HTTPS is enforced** in production.
- CSRF tokens are transmitted via HTTP headers.
- `credentials: "include"` is used to properly control authenticated cross-site requests.

---

> ⚠️ While multiple security controls are implemented, ongoing maintenance, regular audits, and environment-specific configurations are essential to maintaining system security.
